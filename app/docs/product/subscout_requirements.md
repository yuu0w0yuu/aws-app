## 2. プロジェクト概要

### 2.1 プロダクト名
**SubScout（サブスカウト）**

### 2.2 開発目的・解決する課題
多くのユーザーが動画配信サービス、SaaS、ジム、クラウドストレージなどのサブスクリプションを契約していますが、「解約し忘れて意図しない自動更新で料金が発生する」「解約期限が『更新日の3日前まで』『24時間前まで』などサービスごとに異なり、直前に気づいても間に合わない」という課題が存在します。

SubScoutは、単なる契約メモではなく**「解約デッドライン（最終猶予期限）の正確な自動算出」**と**「厳密な状態遷移（ステートマシン）」**をコア機能とし、サブスクの無駄遣いをゼロにするアプリケーションです。

---

## 3. コア機能要件：解約デッドライン算出 ＆ ステートマシン

本プロダクトの競争優位性および学習の核心となるビジネスロジックです。

### 3.1 契約ステータス定義（State Machine）
各サブスクリプションは、必ず以下のいずれかのステータスを保持します。

| ステータスコード | 名称 | 説明 | 次の遷移可能ステータス |
| :--- | :--- | :--- | :--- |
| `TRIALING` | 試用中 | 無料体験期間中。体験終了前に解約しないと自動課金される。 | `ACTIVE`, `CANCEL_PENDING`, `EXPIRED` |
| `ACTIVE` | 契約中（課金中） | 通常の有料契約状態。次回更新日に自動更新される。 | `CANCEL_PENDING`, `EXPIRED` |
| `CANCEL_PENDING` | 解約手続き完了 | ユーザーが解約手続きを完了済。契約終了日まで利用可能だが更新はされない。 | `EXPIRED`, `ACTIVE` (契約再開時) |
| `EXPIRED` | 終了・解約済 | 契約期間が満了し、完全に終了した状態。 | `ACTIVE` (再契約時) |

#### 状態遷移図（概要）
```
 [登録] ──> TRIALING ────(課金開始)────> ACTIVE
           │                               │
           ├─(解約手続き)─┐               ├─(解約手続き)
           │             ▼               ▼
           └───────> CANCEL_PENDING <────┘
                         │
                    (期間満了)
                         ▼
                      EXPIRED
```

### 3.2 解約デッドライン（Deadline）自動算出ロジック
単なる「次回更新日」ではなく、**「自動更新を防ぐために手続きを完了させなければならない最終日時」**を算出します。

#### 算出フォーミュラ・ルール
* **入力パラメータ**:
  * 次回更新日時（`nextRenewalAt`）
  * 解約予告ルール（`cancellationNoticeRule`）:
    * 例: `SAME_DAY`（当日のX時間前まで）
    * 例: `DAYS_BEFORE`（更新日のN日前まで）
    * 例: `HOURS_BEFORE`（更新日時のN時間前まで）
  * 値（`noticeValue`）: N日、またはN時間
  * ユーザーのタイムゾーン（`timeZone`）

* **出力結果**:
  * **解約デッドライン日時（`cancellationDeadlineAt`）**
  * **残り猶予時間カウントダウン（`remainingTime`）**
  * **デッドライン警告レベル（`alertLevel`）**:
    * `SAFE`: 猶予あり（デッドラインまで7日以上）
    * `WARNING`: 注意（デッドラインまで3日以内）
    * `CRITICAL`: 危険（デッドラインまで24時間以内）
    * `EXPIRED`: 期限切れ（デッドライン超過）

## 4. 画面・機能要件

画面はシンプルに保ち、コアロジックの可視化に集中します。

### 4.1 ダッシュボード画面（メイン画面）
* **デッドライン警告ボード**: `CRITICAL`（24時間以内）や`WARNING`のサブスクを最上部に強調表示。
* **ステータス別フィルタリング**: `TRIALING`, `ACTIVE`, `CANCEL_PENDING` ごとの一覧表示。
* **各アイテム表示**:
  * サービス名、アイコン
  * 契約金額 ＆ 更新周期
  * 次回更新日 ＆ **解約デッドライン日時（カウントダウン表示）**
  * 現在のステータスバッジ
  * アクションボタン（「解約完了にする」「ステータス変更」など）

### 4.2 サブスクリプション登録・編集フォーム
* サービス基本情報（名称、カテゴリ、料金、周期）
* 解約ルール設定（「更新日の何日/何時間前までに解約が必要か」を選択）
* 現在のステータス選択（無料体験中 / 通常契約中）

## 5. AWSインフラ・技術学習への展開要素

本Webアプリケーション（Next.js）をAWS上に構築・運用するにあたり、以下の技術要素の学習に繋げます。

1. **ビジネスロジックの単体テスト（Jest / Vitest）**
   * 多様な解約ルールやタイムゾーン、境界値テスト（閏年や月末処理）の網羅。
2. **データベース（Amazon RDS / PostgreSQLなど）**
   * 厳密なステート管理、契約更新・解約履歴ログテーブルの設計。
3. **非同期通知バッチ（Amazon EventBridge + ECS Task / AWS Lambda）**
   * 「解約デッドライン24時間前」のユーザー自動通知処理。
4. **コンテナ化 ＆ インフラ構築（AWS ECS / ALB / Terraform）**
   * Next.jsアプリケーションのECSデプロイとインフラのコード化。
